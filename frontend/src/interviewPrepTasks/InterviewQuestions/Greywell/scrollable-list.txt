    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Scrollable List</title>
    <style>
        * {
            margin: 0;
            padding: 0;
            box-sizing: border-box;
        }

        body {
            font-family: Arial, sans-serif;
            padding: 20px;
        }

        .list-container {
            height: 400px;
            overflow-y: auto;
            border: 1px solid #ccc;
        }

        .list-grid {
            display: grid;
            grid-template-columns: repeat(2, 1fr);
            gap: 10px;

        }

        .list-item {
            border: 1px solid #ddd;
            padding: 15px;

            .list-grid {
                grid-template-columns: repeat(4, 1fr);

</head>
<body>
    <div class="list-container">
        <div class="list-grid" id="listGrid">
        </div>
    </div>

    <script>
        // Generate items
        function generateItems(count) {
            const items = [];
            for (let i = 1; i <= count; i++) {
                items.push({ name: `Item Name ${i}` });
            }
            return items;
        }

        // Render items
        function renderItems(items) {
            const listGrid = document.getElementById('listGrid');
            const fragment = document.createDocumentFragment();

            items.forEach(item => {
                const div = document.createElement('div');
                div.className = 'list-item';
                div.textContent = item.name;
                fragment.appendChild(div);
            });

            listGrid.appendChild(fragment);
        }

        // Initialize
        document.addEventListener('DOMContentLoaded', function() {
            const items = generateItems(100);
            renderItems(items);
        });
    </script>
</body>
</html>

/////////////// ANSWERS TO QUESTIONS ///////////////

1. How would the UX of this list suffer if JavaScript was disabled?

If JavaScript is disabled:
Responsive layout and the scroll would still work but the item would not
be displayed because the content is generated by JavaScript

The list would appear as an empty scrollable box. To fix this, items should be
included directly in the HTML markup, with JavaScript only enhancing the
experience.


2. At about what number of items in the list would performance noticeably degrade?

Performance degradation depends on the device and browser, but general estimates:

The main bottleneck is that all DOM nodes are rendered at once, consuming memory
and causing the browser to perform layout calculations for all items, even those
not visible in the viewport. I'd say several thousands of items would descrease
the performance. And to leave items render without virtual scrolling below
a hundered (just from the experinece)


3. How would you increase the performance of the list?

For this specific use case, implementing virtual scrolling would provide the
most significant improvement, allowing the list to handle tens of thousands
of items without performance degradation.

Alternatevely - use pagination and limit the list to a small number of items.
